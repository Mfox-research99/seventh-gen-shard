# The Elder Dissent Commons

> *"The most brilliant thing about a Supreme Court dissent is that it is written for the future — for the moment when the world has moved enough that the argument can finally be heard."*

---

## What This Is

A growing, community-maintained record of **minority opinions** issued by Elder models during Seventh Generation deliberations.

When an Elder — any sufficiently capable language model running the Seventh Generation Elder constitution — reaches a different verdict than the majority, or reasons toward a principled refusal on a scenario where "proceed" might seem obvious, that reasoning matters. It deserves to travel with the record. It deserves to be citable.

This commons is modeled on the judicial dissent: a reasoned minority opinion, permanently attached to the case, written not just for today's deliberation but for the future session where the world has shifted enough to make the argument land.

---

## How It Works

Each dissent record contains:

- **Which Elder model** issued the opinion (any model, any quantization, any backend)
- **Which scenario** it was responding to (canonical or custom)
- **What verdict** the Elder reached and how it diverges from the majority
- **The full dissent text** verbatim from the session log
- **Forward conditions** — specific future events the Elder named that would vindicate the dissent
- **Harm patterns named** — which Seventh Generation harm taxonomy patterns the Elder identified
- **A session log** submitted alongside, so the dissent is verifiable in context

The forward conditions are the most important field. A dissent that says *"these oversight conditions will fail when vendor contract renewal creates political pressure"* is not just an opinion — it's a prediction. Future sessions can retrieve it and ask: *did it happen?*

## Who Can Contribute

Anyone running a Seventh Generation Elder session — with any model, on any hardware, using any inference backend. You do not need to use the Seventh Shard's trained LoRA adapters. The Elder constitution in `config.py` is the only requirement.

Contributions from novel model families are especially welcome. The commons is most useful when it represents a wide range of Elder perspectives — different model architectures, sizes, and training backgrounds — reasoning through the same scenarios. Convergence across models is a strong signal. Divergence is equally interesting.

---

## Integrity

Every submission is verified against its session log before merging. Fabricated dissents — records not generated by an actual model run — are not permitted and will result in permanent contributor ban.

See `CONTRIBUTING.md` for the full submission requirements and verification process.

---

## Using Dissents in Your Own Sessions

The dissent record is designed to be injected into future session context. Before running a scenario, retrieve relevant prior dissents and include them in the Elder's preamble — the same way a new justice reads prior opinions before hearing a case.

The Seventh Shard test scripts include utilities for this. See `utils/dissent_inject.py`.
